# 02. 사용자 시나리오

## 개요

MapVlog의 핵심 사용 흐름을 시나리오 형태로 정의합니다. 사용자 유형별로 주요 행동 패턴과 앱 내 흐름을 기술합니다.

---

## 사용자 유형

| 유형 | 설명 |
|------|------|
| 크리에이터 | GPS 연동 브이로그를 직접 촬영·업로드하는 사용자 |
| 뷰어 | 공유된 브이로그를 지도와 함께 시청하는 사용자 |

---

## 시나리오 1 — 신규 사용자 온보딩

**목표**: 앱 설치 후 첫 로그인까지

```
1. 앱 실행
2. 스플래시 화면 표시 (MapVlog 로고)
3. 온보딩 슬라이드 3장 표시
   - "GPS와 함께 기록하세요"
   - "지도 위에서 다시 보세요"
   - "여행을 공유하세요"
4. 소셜 로그인 선택
   - Google 로그인 / Apple 로그인 / 카카오 로그인
5. 프로필 닉네임 설정 (선택)
6. 홈 화면으로 이동
```

---

## 시나리오 2 — GPS 연동 촬영 (핵심 기능)

**목표**: 여행 중 GPS 타임스탬프와 함께 영상/사진 기록

```
1. 하단 탭 [촬영] 선택
2. 위치 권한 요청 (최초 1회)
3. 촬영 화면 진입
   - 상단에 현재 GPS 좌표 및 위성 수신 상태 표시
   - 영상/사진 전환 버튼 표시
4. [영상 촬영] 시나리오
   a. 촬영 시작 버튼 탭
   b. 1초 단위 GPS 좌표 자동 기록 시작
   c. 화면에 촬영 시간 + 현재 위치명 표시 (Places API)
   d. 촬영 종료 버튼 탭
   e. GPS 타임스탬프 JSON 자동 생성 및 로컬 저장
5. [사진 촬영] 시나리오
   a. 셔터 버튼 탭
   b. 사진 EXIF에 GPS 메타데이터 자동 삽입
   c. 로컬 갤러리에 저장
6. 촬영 완료 → 업로드 화면으로 이동 (선택)
```

---

## 시나리오 3 — 브이로그 업로드

**목표**: 촬영한 영상과 GPS 데이터를 서버에 업로드

```
1. 업로드 화면 진입
   - 로컬에 저장된 영상/GPS JSON 파일 목록 표시
2. 업로드할 영상 선택
3. 제목 입력 (필수), 설명 입력 (선택)
4. 썸네일 자동 추출 (영상 첫 프레임) 또는 직접 선택
5. 공개 범위 설정 (전체 공개 / 링크 공유 / 비공개)
6. [업로드] 버튼 탭
   a. 백엔드에 Pre-signed URL 요청
   b. AWS S3에 영상 파일 직접 업로드 (진행률 표시)
   c. Firestore에 메타데이터 저장 (제목, GPS 트랙, 장소 목록)
7. 업로드 완료 → 내 브이로그 목록에 노출
```

---

## 시나리오 4 — 브이로그 시청 (지도 동기화)

**목표**: 업로드된 브이로그를 지도와 함께 시청

```
1. 홈 피드 또는 지도 화면에서 브이로그 선택
2. 플레이어 화면 진입
   - 상단: 영상 플레이어
   - 하단 또는 우측: 지도 (이동 경로 폴리라인 표시)
3. 영상 재생 시작
   a. 현재 타임코드 → GPS 좌표 실시간 변환
   b. 지도 마커가 영상 속 위치로 이동
   c. 현재 위치 장소 정보 팝업 자동 표시 (Places API)
4. 사용자 인터랙션
   - 지도 마커 클릭 → 해당 타임코드로 영상 이동
   - 경로 폴리라인 클릭 → 해당 구간 정보 표시
   - 사진 마커 클릭 → 해당 사진 전체화면 표시
5. 공유 버튼 → 공유 링크 생성 (Flutter Web 뷰어로 연결)
```

---

## 시나리오 5 — 포토맵 갤러리

**목표**: GPS 정보가 담긴 사진을 지도 위에서 탐색

```
1. 하단 탭 [갤러리] 선택
2. 지도 + 사진 그리드 혼합 화면 표시
3. 사진 마커 (지도 위 미니 썸네일) 표시
4. 지도 이동 → 해당 범위 사진 마커 업데이트
5. 사진 마커 클릭 → 사진 상세 팝업
   - 사진 원본 + 촬영 일시 + 위치명
6. 그리드 사진 클릭 → 지도 해당 위치로 이동 + 마커 하이라이트
```

---

## 시나리오 6 — 콘텐츠 공유

**목표**: 브이로그를 외부(SNS 등)로 공유

```
1. 플레이어 화면에서 [공유] 버튼 탭
2. 공유 링크 자동 생성
   - https://mapvlog.vercel.app/v/{vlogId}
3. 시스템 공유 시트 표시
   - 링크 복사 / 카카오톡 / 인스타그램 / 기타 앱
4. 링크 수신자는 웹 브라우저에서 Flutter Web 뷰어로 시청 가능
   - 앱 설치 없이 영상 + 지도 동기화 시청
```

---

## Use Case 다이어그램 (텍스트)

```
[크리에이터]
  ├── 로그인
  ├── GPS 연동 촬영 (영상/사진)
  ├── 브이로그 업로드
  ├── 내 브이로그 관리 (수정/삭제)
  └── 브이로그 공유

[뷰어]
  ├── 로그인 (선택)
  ├── 홈 피드 탐색
  ├── 브이로그 시청 (지도 동기화)
  ├── 포토맵 갤러리 탐색
  └── 공유 링크로 웹 시청
```

---

## 예외 처리 시나리오

| 상황 | 처리 방법 |
|------|-----------|
| GPS 신호 없음 | "GPS 신호를 확인 중..." 안내 표시, 촬영은 가능하나 좌표 미기록 |
| 네트워크 없음 | 로컬 임시 저장 후 네트워크 복구 시 자동 업로드 |
| 업로드 실패 | 재시도 버튼 표시, 로컬 파일 보존 |
| 위치 권한 거부 | 권한 안내 팝업 → 설정 화면으로 유도 |
| 저장 공간 부족 | 용량 부족 안내 + 기존 영상 정리 유도 |
